# ComputerAsset 电脑资产管理系统

一个前后端同仓库（Monorepo）的企业资产管理系统，覆盖电脑资产从入库到报废的完整生命周期。

- 仓库地址：<https://github.com/Zzgj/ComputerAsset>
- 技术栈：Vue 3 + Vite + Element Plus + Express + Prisma + SQLite

## 1. 项目结构

```text
.
├─ frontend/                 # 前端（Vue3 + Vite）
├─ backend/                  # 后端 API（Express + Prisma）
├─ README.md                 # 本文档（功能说明 + 运行说明）
└─ .gitignore
```

## 2. 核心功能

### 2.1 资产全流程管理
- 入库（支持模板自动填充）
- 出库（直接领用）
- 分配待领用 / 取消分配 / 确认领用
- 借出 / 归还
- 调拨
- 送修 / 维修完成
- 报废

### 2.2 可视化与审计
- 仪表盘：总量、状态分布、部门分布、提醒通知
- 出入库记录与操作日志
- 资产详情时间线（含历史流转）

### 2.3 系统管理
- 用户管理（super_admin / admin / viewer）
- 部门管理
- 设备模板管理
- 系统配置（如一人一机、提醒天数）
- Excel 导入导出、数据库备份下载
- 设备型号管理显示“关联资产数”（每个模板已关联多少资产）

## 3. 主要状态流转

```text
在库 -> 待领用 -> 使用中 -> 归还 -> 在库
在库 -> 借用中 -> 归还 -> 在库
任意 -> 维修中 -> 在库/已报废
任意 -> 已报废
```

## 4. 权限模型

- `super_admin`：系统全部权限（含用户管理、系统配置、备份）
- `admin`：资产业务管理（入库、出库、借还、调拨、维修等）
- `viewer`：只读查看

### 4.1 权限矩阵（页面/功能）

| 功能 | super_admin | admin | viewer |
|---|---|---|---|
| 仪表盘、资产列表、资产详情、记录查看 | ✅ | ✅ | ✅ |
| 入库、出库/借用、归还、调拨、维修、报废 | ✅ | ✅ | ❌ |
| 部门管理、设备型号管理、导入导出 | ✅ | ✅ | ❌ |
| 用户管理、系统配置、数据备份 | ✅ | ❌ | ❌ |
| 删除资产 | ✅ | ❌ | ❌ |
| 编辑资产关键信息（编号/序列号/品牌等） | ✅（高风险确认） | ❌ | ❌ |

## 6. 本地开发启动

### 6.1 启动后端

```bash
cd backend
# Windows 可手动复制 .env.example 为 .env
pnpm install
pnpm exec prisma migrate deploy
pnpm run dev
```

默认后端地址：`http://127.0.0.1:3000`

### 6.2 启动前端

```bash
cd frontend
pnpm install
pnpm run dev
```

默认前端地址：`http://localhost:5173`

开发模式下前端 `/api` 会代理到后端（默认 `http://127.0.0.1:3000`）。

如果后端端口不同，可在 `frontend/.env.development` 中设置：

```env
VITE_API_PROXY_TARGET=http://127.0.0.1:你的端口
```

## 9. 导入功能规则与限制（重要）

### 9.1 基本规则
- 只读取 Excel 第一个工作表。
- 空白尾行会自动忽略，不计入检测条数。
- 电脑编号必填，缺失会判为无效行并跳过。

### 9.2 模板匹配规则
- 优先按“模板名称”匹配已登记模板。
- 若模板名称未命中，则按“品牌 + 型号”匹配。
- 未登记型号会提示两种处理方式：
  - 创建型号模板并导入
  - 仅导入资产（不创建模板）

### 9.3 状态与人员规则
- 设备状态为 `待领用 / 使用中 / 借用中` 时，`现定人` 必填。
- 设备状态为 `在库` 时，系统会将现定人清空。

### 9.4 序列号规则
- 同一批导入中，序列号不可重复（不同电脑编号冲突会跳过该行）。
- 若序列号与库内其他资产冲突，也会跳过该行。
- 若序列号为空，会自动写入占位值：`暂无-电脑编号`。

### 9.5 日期规则
- 采购日期格式非法：该行无效并跳过。
- 保修到期日格式非法：该行无效并跳过。

### 9.6 预校验与结果展示
- 导入前会先执行预校验，展示：
  - 检测到多少条
  - 预计可导入多少条
  - 无效多少条
- 支持“无效原因分组统计”与“无效行明细（行号 + 原因）”。
- 正式导入时，无效行会被跳过，不影响其他有效行继续导入。
