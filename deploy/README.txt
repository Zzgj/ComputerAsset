============================================================
  ComputerAsset 电脑资产管理系统 v1.2.0
  Windows Server 2019 内网部署指南
============================================================

一、部署前提条件
------------------------------------------------------------
  - Windows Server 2019 Standard 或更高版本
  - Node.js >= 20 已安装 (当前推荐 v22.x)
  - 不需要安装 pnpm、npm 或其他包管理器
  - 不需要外网访问

二、准备部署包（在有网络的机器上执行）
------------------------------------------------------------
  1. 克隆仓库并切换到 deploy/windows-server 分支:

     git clone https://github.com/Zzgj/ComputerAsset.git
     cd ComputerAsset
     git checkout deploy/windows-server

  2. 双击运行 deploy\prepare.bat

     该脚本会自动:
     - 安装前后端依赖
     - 构建前端静态文件
     - 构建后端 JavaScript
     - 生成 Prisma 数据库客户端
     - 打包所有运行时文件到 deploy-package\ 文件夹

  3. 将 deploy-package 文件夹整体拷贝到内网服务器
     (U盘、文件共享等方式均可)

三、在服务器上部署
------------------------------------------------------------
  1. 将 deploy-package 文件夹放置到目标路径
     例如: D:\ComputerAsset\

  2. 双击运行 deploy.bat

     该脚本会自动:
     - 检查 Node.js 版本
     - 验证所有文件完整性
     - 创建 .env 配置文件（首次运行）
     - 创建数据目录
     - 执行数据库迁移
     - 检查端口冲突（支持自动终止占用进程）
     - 启动服务

  3. 看到 "部署完成" 提示后，使用浏览器访问:

     本机: http://127.0.0.1:3000
     局域网: http://服务器IP:3000

五、配置说明
------------------------------------------------------------
  配置文件位于 backend\.env

  PORT=3000              服务端口号
  DATABASE_URL=...       SQLite 数据库路径
  JWT_SECRET=...         JWT 签名密钥（务必修改为随机字符串）
  JWT_EXPIRES_IN=24h     登录过期时间
  BACKUP_DIR=backup      备份目录

  修改端口后需重启服务生效。

六、日常运维
------------------------------------------------------------
  启动服务: 双击 deploy.bat
  停止服务: 双击 stop.bat 或关闭服务命令行窗口
  查看日志: 直接查看服务命令行窗口输出
  数据备份: 在系统内使用"数据备份"功能,或直接复制
            backend\data\dev.db 文件
  数据恢复: 停止服务后替换 backend\data\dev.db 文件,
            再重新启动

七、防火墙设置（如果局域网其他电脑需要访问）
------------------------------------------------------------
  在 Windows 防火墙中放行入站规则:

  1. 打开"Windows Defender 防火墙" -> "高级设置"
  2. 入站规则 -> 新建规则
  3. 选择"端口" -> TCP -> 特定本地端口: 3000
  4. 允许连接
  5. 选择适用的网络配置文件
  6. 命名为 "ComputerAsset"

  或使用命令行:
  netsh advfirewall firewall add rule name="ComputerAsset" ^
    dir=in action=allow protocol=tcp localport=3000

八、常见问题
------------------------------------------------------------
  Q: 启动时提示端口被占用
  A: deploy.bat 会提示终止占用进程,选 y 即可;
     或修改 backend\.env 中的 PORT 为其他端口号

  Q: 访问时页面空白
  A: 确认 frontend\dist\ 文件夹存在且包含 index.html

  Q: 登录失败
  A: 首次部署需等待数据库迁移完成;
     检查 backend\.env 中 JWT_SECRET 是否配置

  Q: 需要更新版本
  A: 在有网络的机器上重新运行 prepare.bat,
     将新的 deploy-package 覆盖到服务器, 再运行 deploy.bat
